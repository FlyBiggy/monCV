Chasse au Monstre

Description :

Le projet "Chasse au Monstre" consiste à développer un jeu en JavaFX pour deux joueurs ou plus. Le jeu se déroule dans un labyrinthe où un chasseur doit débusquer un monstre. Le chasseur ne voit pas le monstre et doit le retrouver en tirant sur des cases du labyrinthe. Le monstre, quant à lui, connait le labyrinthe et doit atteindre la sortie sans se faire attraper.

Livrables

Le projet est découpé en deux livrables.

Livrable 1

  • Version simple du jeu jouable par deux joueurs humains.
  • Code source, tests unitaires et documentation.

Livrable 2

  • Version aboutie du jeu avec des fonctionnalités avancées :
    • IA pour le chasseur et le monstre
    • Labyrinthes aléatoires et personnalisables
    • Choix des paramètres du jeu
    • Inter-opérabilité des IA

Compétences évaluées

  • Programmation Java
  • Conception logicielle
  • UML
  • Javadoc
  • Organisation et travail en équipe

Points importants

  • Respecter les spécifications du document.
  • Travail régulier et équitable de tous les membres du groupe.
  • Prioriser les fonctionnalités requises.

SAE pizzaland

Description :

Gestion de Pizzeria avec API REST

Ce projet décrit le développement d'une API RESTful en Java pour gérer les commandes et les pizzas d'une pizzeria. Nous allons construire cette API sans frameworks supplémentaires, en utilisant uniquement des technologies standards.

Technologies utilisées

  • Java: Langage de programmation pour le développement de l'API
  • Tomcat: Serveur web pour héberger l'API
  • PostgreSQL: Base de données pour stocker les informations sur les ingrédients, pizzas et commandes
  • Java Servlets: Technologie Java pour créer des applications web dynamiques, utilisées pour gérer les requêtes HTTP et les réponses de l'API

Fonctionnalités

  • Gestion des ingrédients:
    • Ajout, modification et suppression d'ingrédients
    • Consultation de la liste des ingrédients disponibles
  • Gestion des pizzas:
    • Ajout, modification et suppression de pizzas
    • Consultation de la liste des pizzas avec leurs ingrédients
  • Gestion des commandes:
    • Ajout, modification et suppression de commandes
    • Consultation de l'historique des commandes
  • Authentification:
    • Utilisation d'un token pour l'identification des utilisateurs autorisés
    • Contrôle d'accès aux différentes ressources de l'API

Avantages

  • Simplicité:

    • L'implémentation de l'API est plus basique et facile à comprendre, idéale pour apprendre les concepts fondamentaux des APIs REST.

  • Légereté:

    • L'application est légère et consomme peu de ressources serveur, adaptée aux petits projets.

  • Apprentissage:

    • Ce projet permet d'apprendre les technologies Java Servlets et l'accès aux bases de données pour la construction d'APIs REST.

Structure du projet

  • Modèles (Java):

    • Classes Java représentant les entités du domaine (ingrédients, pizzas, commandes, etc.)

  • Servlets (Java):

    • Classes Java qui gèrent les requêtes HTTP (GET, POST, PUT, DELETE) envoyées par les clients et renvoient les réponses correspondantes.

  • DAO (Data Access Objects) - Java:

    • Classes Java responsables de l'interaction avec la base de données PostgreSQL. Elles gèrent les requêtes SQL pour ajouter, modifier, supprimer et récupérer des informations.

  • Fichiers de configuration:

    • Fichiers de configuration pour Tomcat et la base de données PostgreSQL.

Développer une API REST pour une pizzeria avec Java, Tomcat et PostgreSQL est une approche simple et efficace pour créer une application web légère et performante. Cela permet aux développeurs d'apprendre les concepts clés des APIs REST et de se familiariser avec les technologies Java standard pour la construction de services web.

Description :

Objectif

Développer un jeu en ligne multi-joueur de type "shoot them up" en équipe de 3 étudiants.

Fonctionnalités

    • Accueil/Connexion
